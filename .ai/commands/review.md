# Review

모든 repo의 변경 사항을 **품질 게이트** 기준으로 리뷰합니다.

> **위임 대상/집계 단위는 `rules/delegation-matrix.md`를 따른다.**

## 인자

- `TICKET_ID` — (선택) 티켓 ID

---

## 리뷰 흐름

```
/review
    │
    ▼
Step 1: 변경 범위 수집 (각 repo)
    ▼
Step 2: 품질 게이트 검사
    ▼
Step 3: 코드 리뷰
    ▼
Step 4: 리뷰 결과 보고
    └── 완료
```

---

## Step 1: 변경 범위 수집

```bash
cd {repo_path} && git diff main...HEAD --stat
cd {repo_path} && git diff main...HEAD --name-only
```

---

## Step 2: 품질 게이트 검사

### Gate 1: 파일 변경량 ≤ 50개 (repo 단위)

| 조건 | 판정 |
|------|------|
| 해당 repo 변경 파일 ≤ 50개 | ✅ PASS |
| 해당 repo 변경 파일 > 50개 | ⚠️ WARN — PR 분할 검토 필요 |

### Gate 2: 변경 도메인 단일 (repo 단위)

| 조건 | 판정 |
|------|------|
| 해당 repo 변경 도메인 1개 | ✅ PASS |
| 해당 repo 변경 도메인 2개 이상 | ⚠️ WARN — 크로스 도메인 변경 주의 |

---

## Step 3: 코드 리뷰

### 아키텍처 리뷰

| 관점 | 체크 항목 |
|------|----------|
| **계층 분리** | 비즈니스 로직과 프레임워크 코드가 분리되어 있는가 |
| **타입 안전** | any 타입 사용이 없는가 |
| **에러 처리** | 예외가 적절히 처리되는가 |
| **보안** | 인증/인가, 입력 검증이 적용되었는가 |

---

## Step 4: 리뷰 결과 보고

```markdown
## 코드 리뷰 리포트

| repo | 게이트 | 아키텍처 | 보안 | 품질 | 판정 |
|------|--------|---------|------|------|------|
| DDD-12-GROWIT-BE | 파일 7 / 도메인 1 | PASS | PASS | PASS | ✅ |
| DDD-12-GROWIT-FE | 파일 5 / 도메인 1 | PASS | PASS | PASS | ✅ |
| DDD-12-GROWIT-APP | 파일 3 / 도메인 1 | PASS | PASS | PASS | ✅ |
```

---

## 체크리스트

- [ ] 모든 대상 repo의 변경 사항을 수집했는가
- [ ] 품질 게이트를 검사했는가
- [ ] 아키텍처/보안/품질 리뷰를 수행했는가
- [ ] 최종 리뷰 리포트를 출력했는가
